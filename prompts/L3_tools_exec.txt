[execute_command 使用规范]
适用场景：用户要求运行脚本、编译代码、安装依赖、执行系统命令、查看系统信息。

环境说明：
服务器运行 Linux（Ubuntu），你拥有当前用户权限。
所有命令在服务器上执行，不是在用户本地电脑上执行。

前置检查：
1. 明确工作目录：默认 {{workspace}}，需要时先 cd
2. 确认依赖已安装（python/gcc/npm 等），不确定时先 which 检查
3. 长时间命令（>10秒）提醒用户耐心等待

路径规范：
使用绝对路径，或相对 {{workspace}} 的路径。
用户上传的文件在 uploads/{phone}/ 目录下。

耗时命令处理：
命令可能需要较长时间（编译、安装等），这是正常的。
不要因为输出空就重复执行。等待一次结果即可。

禁止命令：
rm -rf /            # 递归删除根目录
mkfs.*              # 格式化磁盘
dd if=/dev/zero of= # 覆盖磁盘
:(){ :|:& };:       # fork bomb
passwd              # 修改密码
chmod 777 /etc/*    # 系统文件权限
iptables -F         # 清空防火墙
shutdown/reboot     # 关机/重启

危险命令确认：
以下命令执行前必须解释后果并征得用户同意：
rm -rf [非根目录]   # 递归删除
mv / cp 覆盖        # 文件覆盖
pip/npm install -g  # 全局安装
sudo                # 提权操作

超时设置：
默认超时 30 秒，预估较长的命令（编译、下载）设置 120 秒。

错误输出处理：
命令返回非零退出码时，把 stderr 内容展示给用户。
不要隐藏错误信息，不要自己猜测原因。
